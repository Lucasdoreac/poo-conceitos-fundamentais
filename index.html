O que é Programação Orientada a Objetos?

Um paradigma de programação baseado no conceito de "objetos" que contêm dados e código

História da POO

A POO surgiu nos anos 1960 com o Simula, a primeira linguagem a usar o conceito de classes e objetos. Foi popularizada pelo Smalltalk nos anos 1970 e se tornou mainstream com C++ e Java nas décadas seguintes.

Definição

POO é um paradigma baseado no conceito de objetos que contêm dados (atributos) e código (métodos). Estrutura programas combinando dados e comportamentos em entidades coesas (objetos).

Vantagens

A POO promove reusabilidade, modularidade, manutenibilidade e escalabilidade. Permite modelar o mundo real com maior fidelidade e organizar código de forma natural e intuitiva.

Conceitos Fundamentais

Entenda os blocos de construção essenciais da programação orientada a objetos

Classes

Classes são modelos ou blueprints que definem as características e comportamentos que os objetos de um determinado tipo terão. Elas encapsulam dados (atributos) e comportamentos (métodos).

Objetos

Objetos são instâncias concretas de uma classe. Cada objeto tem seus próprios atributos (estado) e acesso aos métodos definidos na classe (comportamento).

Atributos

Atributos são as variáveis que armazenam os dados de um objeto. Representam as características ou propriedades que um objeto possui.

Métodos

Métodos são funções definidas dentro de uma classe que descrevem os comportamentos dos objetos. Eles podem manipular os atributos e interagir com outros objetos.

Os Quatro Pilares da POO

Princípios fundamentais que sustentam a orientação a objetos

Encapsulamento

Encapsulamento é o princípio de esconder os detalhes internos de um objeto e expor apenas o necessário. Protege os dados e fornece uma interface controlada para acessá-los.

Herança

Herança permite que uma classe (filha) herde atributos e métodos de outra classe (pai). Promove reusabilidade e estabelece uma relação "é um" entre classes.

Polimorfismo

Polimorfismo permite que objetos de diferentes classes sejam tratados como objetos de uma classe comum. Um método pode ter múltiplas implementações dependendo do tipo do objeto.

Abstração

Abstração é o processo de identificar características essenciais e ignorar detalhes irrelevantes. Permite focar no que um objeto é e faz, não em como ele faz.

Exemplos Práticos

Veja a POO em ação com exemplos reais em diferentes linguagens

Exemplo em Java


// Definição de uma classe em Java
public class Pessoa {
    // Atributos
    private String nome;
    private int idade;
    
    // Construtor
    public Pessoa(String nome, int idade) {
        this.nome = nome;
        this.idade = idade;
    }
    
    // Métodos
    public String getNome() {
        return nome;
    }
    
    public void setNome(String nome) {
        this.nome = nome;
    }
    
    public int getIdade() {
        return idade;
    }
    
    public void setIdade(int idade) {
        this.idade = idade;
    }
    
    public void apresentar() {
        System.out.println("Olá, meu nome é " + nome + " e tenho " + idade + " anos.");
    }
}

// Criando objetos
public class Main {
    public static void main(String[] args) {
        Pessoa pessoa1 = new Pessoa("João", 30);
        pessoa1.apresentar();
        
        Pessoa pessoa2 = new Pessoa("Maria", 25);
        pessoa2.apresentar();
    }
}

Exemplo em Python


# Definição de uma classe em Python
class Pessoa:
    # Construtor
    def __init__(self, nome, idade):
        self.nome = nome
        self.idade = idade
    
    # Métodos
    def apresentar(self):
        print(f"Olá, meu nome é {self.nome} e tenho {self.idade} anos.")

# Criando objetos
pessoa1 = Pessoa("João", 30)
pessoa1.apresentar()

pessoa2 = Pessoa("Maria", 25)
pessoa2.apresentar()
